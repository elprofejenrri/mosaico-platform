# Roadmap

## Phase 1: Stabilize The Platform Foundation

Status: in progress.

- Production-safe auth and session rules.
- Standardized database schema.
- RBAC roles, permissions, and user-role assignment.
- Admin user management UI.
- Super Admin configuration center.
- Audit logs, activity logs, analytics events, and request IDs.
- Technical wiki visible in the product.

## Phase 2: Convert Education Workflows To Real Backend Flows

- Student learning roadmap persistence.
- Badge and level unlock rules.
- Practice tests and formal tests.
- Credit wallet ledger and credit grants.
- Class booking/rescheduling/cancellation endpoints.
- Teacher feedback and class completion flows.
- Parent/tutor multi-student account management.

## Phase 3: Calendar And Scheduling Engine

- Google Calendar OAuth connection.
- Teacher availability as backend source of truth.
- Slot generation with 30, 45, and 60 minute options.
- Cooldown gap rules.
- Conflict detection and imported busy blocks.
- Student booking conversion analytics.

## Phase 4: Launch Operations

- Alpha with internal users.
- Beta with invited students and teachers.
- Production with backups, incident runbook, privacy terms, support workflow, and rollback process.

## Next Recommended Sprint

Build the credit ledger and production booking lifecycle. These are the highest-value launch blockers because they touch money, scheduling, student trust, teacher operations, and analytics.
