Product Summary
===============

Mosaico is an online Spanish learning platform with multiple user personas:

1. Client / Student
   - The learner.
   - Needs a roadmap of learning, badges, levels, tests, practice tests, credits, unlocked progress, classes, AI/practice support, community moments, and progress visibility.

2. Tutor / Parent
   - A guardian, parent, or tutor who can manage one or more students.
   - Needs family wallet/credits, student progress, bookings, feedback, alerts, reports, communication with teachers, and visibility into learning milestones.

3. Teacher
   - Delivers classes and manages availability.
   - Needs calendar, availability, blocked time, booked/empty slots, class outcomes, students, materials, evaluations, feedback, earnings, and Google Calendar integration.

4. Administrative / Principal
   - Non-technical school administrator.
   - Can approve users, give credits, create lessons, review teachers, operate bookings and reports, manage school-side processes.
   - Current canonical role: administrador_escolar.
   - Current portal route: /school-admin.
   - Product label: School Administrative / Administrativo escolar.

5. Super Admin / Technical Admin
   - Technical and governance profile.
   - Can manage platform configuration, IAM/RBAC, audit logs, activity logs, technical wiki, Mosaico Atlas, system health, analytics, and sensitive settings.
   - Super Admin canonical role: administrador_sitio.
   - Technical Admin canonical role: administrador_profesor.
   - Product label for technical workspace: Technical Admin / Admin tecnico.

Teacher Google Calendar Local Implementation
============================================

- Separate per-teacher Calendar OAuth; Supabase login tokens and the legacy
  shared admin calendar are not reused.
- Backend Authorization Code Flow uses signed, expiring, one-use state.
- Access and refresh tokens are encrypted with a configured Fernet key.
- Calendar IDs are verified against Google; busy sources may be read-only and
  the single class-event destination must be owned.
- Only free/busy intervals are cached; personal event content is not imported.
- Availability/booking checks fail closed on provider uncertainty or conflict.
- MOSAICO event synchronization uses deterministic IDs and persisted retry state.
- Disconnect preserves MOSAICO classes and existing Google events.
- Optional cron entry: backend/sync_google_calendars.py.
- Additive migration: backend/migrations/004_google_calendar_teacher_integration.sql.
- Contract and setup: docs/GOOGLE_CALENDAR_INTEGRATION.md.
- Backend switch GOOGLE_CALENDAR_INTEGRATION_ENABLED and platform flag
  teacher_google_calendar both default to false.
- The work remains local until `magic`: no release entry, commit, push,
  migration, deployment, backfill, or production flag change has occurred.
